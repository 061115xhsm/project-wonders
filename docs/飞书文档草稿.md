## 一、项目简介

**万泰新天地智能运营中台** 是面向普宁最大商业综合体(广东万泰集团·万泰汇购物中心,20万㎡,2014开业,5层)的 **AI 驱动运营决策系统**。

系统直击万泰在黑客松企业调研中提出的核心痛点:

1. **不知来商场的人是谁** —— 客流画像缺失,消费习惯不明
2. **线上线下会员不打通** —— 营销割裂
3. **会员拉新靠传统方式** —— 数字化获客不知怎么做
4. **商铺经营数据采集不到** —— 客流、销售数据缺失
5. **租户管理繁琐** —— 收租提醒靠人工

我们用 **RAG 知识库 + 大模型营销 Agent** 把商场从"凭经验拍脑袋"转向"看数据做决策",在客流画像、会员分层、商铺评估、智能营销四大场景形成闭环。

**一句话介绍**:让普宁最大的商场,拥有一个能看懂客流、读懂会员、给得出营销方案的 AI 运营大脑。

## 三、AI 创新性说明(核心)

> 评审硬门槛:≥2 项 AI 原生能力(大模型/多模态/Agent/RAG/AI 工作流)深度绑定场景。
> 本项目深度组合使用 **RAG + Agent + 大模型** 三项 AI 原生能力,与万泰运营场景深度绑定。

### 3.1 RAG 知识库(检索增强生成)

不是简单调大模型,而是先 **构建商场专属知识库**(94 条语料,4 类):
- **真实商户清单**:60 个品牌的业态/楼层/主力店/营销定位(如"喜茶→高频到店复购驱动→适合券+会员日")
- **万泰企业资料**:企业简介/5 大痛点/可用数据/合规边界(来自企业弹药库)
- **营销方法论**:RFM 四类群体的标准策略模板 + 商铺评分公式 + ROI 预估方法
- **业务指标体系**:客流/销售额/转化率/租售比/分群/等级的指标释义

用 **sentence-transformers 向量检索**(多语言 MiniLM 模型),Agent 提问时先语义检索 top-k 相关知识,再注入大模型 context。这让大模型回答 **基于商场真实数据与方法论,而非凭空生成**——选品推荐只从真实商户里选(如沉睡会员推 NIKE/喜茶/沃尔玛),不编造。

### 3.2 营销 Agent(AI 工作流核心)

`5_营销助手` 页是评审看的 AI 工作流:

```
选目标 RFM 群体 → 取该群体真实统计(人数/均消费/均到店/均客单)
  → 工具①:RAG 检索商户清单 + 营销方法论(语义检索)
  → 工具②:大模型基于 RAG context + 真实数据生成结构化方案
  → 输出 JSON:营销目标 / 3条策略(含触达渠道) / 联动商户推荐 / 投放话术 / ROI预估 / 合规提示
  → 分块卡片渲染
```

输出可直接落地的方案,例如沉睡群体:"满300立减60,7天内到店送喜茶饮品券",ROI 预估 10x,联动商户 NIKE/喜茶/沃尔玛(均来自知识库真实品牌,附推荐理由)。

### 3.3 AI 运营日报(首页)

首页 Agent 基于全局真实统计自动生成 3 条可行动洞察,如:"VIP占比仅1.6%(78人),建议定向推送高额满减券提升高净值转化"——含具体数字 + 建议动作。

### 3.4 为什么是 AI 原生而非传统系统

- 传统 BI 只展示数据;**本系统让大模型读懂数据 + 检索知识 + 给出方案**,形成"数据→洞察→决策→执行"闭环
- 规则引擎只能给固定文案;**Agent 基于真实群体统计动态生成**,不同群体/不同数据出不同方案
- **双模型 fallback**:讯飞 maas-coding(auto,200k 上下文)首选,智多云(glm-5.2/deepseek-v4/kimi)兜底,大模型不可用时自动降级规则引擎,保证 Demo 不崩

## 四、技术实现说明

### 4.1 技术栈

| 层 | 技术 |
|---|---|
| 前端 | Streamlit + Plotly(暗色设计大屏,自定义 CSS 组件) |
| AI | 大模型(讯飞/智多云 OpenAI 兼容)+ sentence-transformers 向量检索 |
| 算法 | scikit-learn(RFM + K-Means 聚类 + 商铺评分) |
| 数据 | pandas + numpy(ETL + 特征工程) |

### 4.2 核心算法

**RFM 会员分层**:R=最近到店天数、F=到店次数、M=累计消费。K-Means(K=4)聚类后按聚类中心综合评分(-R+F+M)降序 **动态映射标签,不硬编码簇编号**,输出高价值/潜力/新客/沉睡。

**商铺综合评分**:`Score = 0.4×销售 + 0.3×客流 + 0.2×租售比 + 0.1×转化率`,各指标 Min-Max 归一化。

**RAG 检索**:语料向量化(384维)→ L2 归一化 → 余弦相似度 top-k → 注入大模型 context。

### 4.3 数据安全

- 手机号脱敏(138****5678)、姓名脱敏(王**)、身份证不存储
- 数据文件不进 Git(`.gitignore`)
- AI 输出含合规提示

## 五、交互与设计说明

### 5.1 信息架构(运营决策流)

从"平铺 BI 看板"改为"运营决策流",5 个页面层层递进:

| 页面 | 叙事 | 核心 |
|---|---|---|
| 首页·运营全景 | 一屏看清现状 | Hero营收 + KPI + **AI运营日报** + 客流趋势 + Top5 |
| 客流·时空洞察 | 人从哪来、何时来 | 7×24热力图 + 天气/节假日效应 |
| 会员·分层运营 | 人是谁、怎么分 | RFM分层 + 等级分布 + 群体消费对比 |
| 商铺·经营评估 | 谁在赚钱、谁有盲区 | 真实商户排行(可筛选) + 业态雷达 + 楼层评分 + 坪效散点 |
| 营销·AI助手 | 该怎么动、效果几何 | 选群体 → **Agent生成方案** → 投放话术 → ROI预估 |

### 5.2 设计系统

- 暗色专业大屏(背景 #0d0d0d / 卡片 #1a1a19),金色 #c98500 主强调(贴合万泰调性)
- 固定分类色(颜色跟随实体不跟随排名,切换筛选不重涂),已过 CVD 色觉障碍验证
- 统一标记规范:线 2px、柱≤24px、marker≥8px 带 surface ring、网格 1px hairline
- KPI 卡片:左侧强调色条 + tabular-nums(数字等宽对齐)
- AI 输出组件:营销方案分块卡(策略/选品/话术/ROI 网格)+ "AI 生成/规则兜底"来源徽章

### 5.3 关键交互

- **营销页**:选群体 + "启用 AI 生成"开关(可对比 AI vs 规则),Agent 秒级出方案
- **商铺页**:楼层/业态/主力店三维筛选,65 行真实商户可快速定位
- **侧边栏**:AI 引擎就绪状态指示灯(绿=大模型+RAG 就绪)
