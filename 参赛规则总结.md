## 二、提交物要求

### 2.1 海报（必交）

| 项目 | 要求 |
|------|------|
| 格式 | PDF |
| 尺寸 | 297cm(宽) × 420cm(高)，竖版，宽高比1:1.414 |
| 分辨率 | 150 DPI |
| 色彩模式 | CMYK |
| 命名 | `[赛道]_[队伍名称]_[作品名称]海报.pdf` |
| 内容 | 项目名称、一句话介绍、核心能力/亮点、Demo方式或二维码 |
| DDL | **7月11日 20:30** |
| 注意 | 超时未提交→自行准备海报；图片需高清；重要内容距边缘≥1cm |

### 2.2 飞书文档链接（必交）

文档必须包含：
1. **项目简介**（解决什么问题）
2. **业务价值说明**（对用户或业务的实际意义）
3. **AI创新性说明**（AI在哪里、为什么是核心）
4. **技术实现说明**（整体思路即可，不要求源码细节）
5. **交互与设计说明**（关键流程或界面）

要求：图文并茂、逻辑清晰、**提前设置分享权限确保评委可访问**

### 2.3 演示视频MP4（必交）

| 项目 | 要求 |
|------|------|
| 时长 | **3分钟以内** |
| 内容 | 产品Demo，真实录屏/实机操作 |
| 重点 | 清晰展示核心功能及AI能力 |
| 不需要 | 专业包装、特效剪辑 |

## 四、评审维度（满分100分）

### 4.1 通用评审表（所有赛道）

| # | 维度 | 权重 | 核心考察点 |
|---|------|------|-----------|
| 1 | **场景契合度与行业价值** | 25% | 与赛道核心定位、普宁本土场景的匹配度，是否结合企业参访/民俗体验调研成果 |
| 2 | **方案可行性与落地能力** | 25% | 方案可实施性、部署难度、运行稳定性，是否可对接本地企业落地 |
| 3 | **产品/方案完整度与体验** | 20% | 核心功能完整度、用户体验、业务/场景闭环能力 |
| 4 | **技术/创意创新性** | 15% | 是否基于AI原生能力（大模型/多模态/Agent/RAG/AI工作流），而非传统系统开发 |
| 5 | **商业/规模化潜力** | 15% | 市场空间、商业模式、规模化复制能力、对普宁产业/文化的带动效应 |

### 4.2 AI+软件赛道打分细则

| 维度 | 软件赛道侧重 |
|------|------------|
| 场景契合度 | 普宁产业数字化痛点挖掘深度、业务流程适配性 |
| 方案可行性 | 系统兼容性、与现有产业系统对接难度、运行稳定性 |
| 产品完整度 | 功能完整度、产业用户体验、业务闭环能力 |
| 技术创新性 | 数字化模式创新、AI算法/架构创新、AI工作流对产业流程的优化效果 |
| 商业潜力 | 产业市场规模、商业模式、规模化复制能力 |

### 4.3 技术创新性评分阶梯（最关键维度）

| 分数 | 标准 |
|------|------|
| **12-15分** | 深度融合AI原生能力，创新组合使用**大模型/多模态/Agent/RAG/AI工作流中2项及以上**，与赛道场景深度绑定，形成传统开发无法实现的差异化创新 |
| **8-11分** | 成熟应用AI原生能力，使用上述1项及以上核心能力，适配赛道场景需求，具备清晰创新点 |
| **4-7分** | ⚠️ 仅简单调用通用大模型接口，无深度场景融合，整体与传统系统开发水平相当 |
| **0-3分** | ❌ 无AI原生能力应用，为传统系统开发方案 |
